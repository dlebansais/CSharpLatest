# CSL1014: Consider using `<inheritdoc />`

Diagnoses when some code documentation is duplicated.

Consider the following declarations, where a class implements an interface:

```cs
class Foo : IDisposable
{
    /// <summary>
    /// Performs application-defined tasks associated with freeing, releasing, or resetting unmanaged resources.
    /// </summary>
    public void Dispose()
    {
        /* ... */
    }
}
```

The summary is a copy of the `IDisposable.Dispose` doc. It may be preferable to use the `inheritdoc` tag instead:

```cs
class Foo : IDisposable
{
    /// <inheritdoc />
    public void Dispose()
    {
        /* ... */
    }
}
```

The CSL1014 diagnostic is also issued when overriding a virtual method.

See the [inheritdoc](https://learn.microsoft.com/en-us/dotnet/csharp/language-reference/xmldoc/recommended-tags#inheritdoc) section for replacing or overriding documentation.

## Sample code

```cs
class Foo : IDisposable
{
    /// <summary>
    /// Dispose of resources.
    /// </summary>
    public void Dispose() // CSL1014: Consider using the <inheritdoc/> tag to document 'Dispose'
    {
        /* ... */
    }
}
```
